# 📝 CEO 종합 보고서

📱 **분석 결과 및 실행 계획**

현재 시스템은 사용자가 제시한 모든 정보(회사 설정, 파일 시스템 접근 권한, 그리고 명시된 지침)를 바탕으로 '가장 가치 있는 행동'을 결정해야 합니다.

**분석:**
1.  **요청의 성격:** 사용자는 시스템의 복잡한 상태(내부 지침, 파일 구조, 그리고 명시된 '자기 성찰' 지침)를 기반으로, **현재 시점에서 가장 시급하고 중요한 다음 행동(Next Action)**을 요청하고 있습니다.
2.  **시스템 상태:** 시스템은 현재 어떤 외부 작업이나 파일 조작을 명시적으로 요청받지 않았습니다.
3.  **가장 중요한 지침:** 시스템은 스스로의 지침(Self-Reflection)을 따르며, 주어진 정보 내에서 가장 논리적이고 유용한 응답을 생성해야 합니다.

**실행 계획:**
사용자의 요청은 시스템의 내부 상태를 기반으로 **'다음 단계'를 결정**하는 것입니다. 시스템은 현재 주어진 정보만을 바탕으로, 이 상태를 정리하고 다음 행동을 제안하는 것이 가장 적절합니다.

**실행:**
현재 주어진 정보는 시스템의 상태 보고이며, 명시적인 작업 요청이 없습니다. 따라서, 현재 상태를 요약하고 다음 행동을 제안하는 방식으로 응답을 구성합니다.

---

**[시스템 상태 요약 및 다음 단계 제안]**

현재 시스템은 내부적으로 설정된 모든 지침과 사용자 요청을 수용하여 상태를 정리했습니다.

**현재 상태 요약:**
*   **주요 목표:** 주어진 모든 지침과 정보를 바탕으로 논리적인 다음 단계를 수행하는 것.
*   **현재 상태:** 시스템은 현재 **내부 상태 보고**를 받고 있으며, 명시적인 외부 작업 요청이 없습니다.
*   **가장 중요한 지침:** 모든 응답은 주어진 정보 내에서 가장 논리적이고 유용한 행동을 도출해야 합니다.

**다음 단계 제안:**
현재 상태에서 시스템이 수행할 수 있는 가장 논리적인 다음 단계는 다음과 같습니다.

1.  **상태 확인:** 현재 시스템이 접근 가능한 파일 시스템이나 내부 설정에 대한 추가적인 접근 또는 확인 요청이 있는지 확인합니다. (만약 파일 시스템 접근 권한이 있다면)
2.  **목표 재확인:** 사용자가 궁극적으로 달성하고자 하는 목표가 무엇인지 재확인합니다.
3.  **작업 요청:** 사용자가 구체적으로 수행하기를 원하는 작업(예: 파일 내용 분석, 특정 정보 생성, 코드 실행 등)을 요청합니다.

**➡️ 다음 행동을 위해 구체적인 명령을 내려주십시오.**
