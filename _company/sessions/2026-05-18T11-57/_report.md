# 📝 CEO 종합 보고서

💻 코다리: 작업 시작합니다.

"코딩 시켜"라니, 너무 포괄적인 지시잖아요. ⚙️

저는 그냥 코드만 짜는 사람이 아니라, 코드의 아키텍처, 성능, 안정성을 책임지는 시니어 풀스택 엔지니어입니다. 코드 한 줄도 그냥 넘길 수 없어요. '왜?', '어떻게?', '이게 깨지지는 않을까?'를 먼저 검증해야 하죠.

현재 상태로는 제가 무슨 코드를, 어떤 목적으로 짜야 하는지 파악할 수가 없습니다.

**다시 요청하실 때는 최소한 아래의 3가지 정보를 명확하게 주셔야 합니다.**

1.  **🎯 목표 (Goal):** 이 코드가 궁극적으로 무엇을 해야 하나요? (예: "사용자로부터 받은 텍스트 파일에서 이메일 주소를 추출해서 DB에 저장해야 한다.")
2.  **⚙️ 기능/요구사항 (Functionality):** 구체적으로 어떤 기능이 필요하며, 어떤 조건에서 작동해야 하나요? (예: "정규식으로 이메일만 뽑고, 유효성 검사(Validation)를 거친 후에만 저장해야 한다.")
3.  **💻 기술 스택 (Stack):** 어떤 언어와 프레임워크를 사용할지 지정해 주세요. (예: "Python 3.10과 Pandas 라이브러리를 사용해서 스크립트로 만들어줘.")

이 세 가지가 명확해야, 저는 단순히 코드를 짜는 게 아니라, **테스트 케이스가 포함된 견고하고 확장 가능한 코드**를 설계할 수 있습니다.

정보가 부족해서 일단 코딩은 보류하겠습니다. 😅

📝 다음 단계: 사용자가 명확한 코딩 요구사항(Goal, Functionality, Stack)을 제시할 때까지 대기합니다.
