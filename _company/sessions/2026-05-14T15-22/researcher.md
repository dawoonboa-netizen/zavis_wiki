## ⚾ KBO 경기 예측 웹사이트 데이터 수집 및 구조화 보고서

### 1. 프로젝트 목표 재정의 및 데이터 요구사항 분석

**최종 목표:** 단순히 과거 기록을 나열하는 것이 아니라, **승패 예측의 근거가 되는 가중치 기반의 통계 데이터**를 제공하는 것입니다.

**필요 데이터 유형:**
1. **팀 성능 지표 (Team Performance Metrics):** 전반적인 전력 측정.
2. **상대 전적 지표 (Head-to-Head Metrics):** 특정 팀 간의 심리적 우위 분석.
3. **상황 변수 지표 (Contextual Variables):** 경기 환경 및 선수 컨디션 반영.

---

### 2. 핵심 데이터 구조 설계 (Data Schema)

아래의 표는 백엔드 데이터베이스에 저장되어야 할 핵심 데이터를 정의합니다.

| 데이터 항목 (Field Name) | 데이터 유형 | 설명 및 측정 방식 | 데이터 출처 (Source) | 가중치 적용 여부 |
| :--- | :--- | :--- | :--- | :--- |
| **경기 ID (Game ID)** | String | 고유 식별자 (예: 20240515_A_B) | API/수동 입력 | N/A |
| **홈팀 (Home Team)** | String | 경기장 홈 팀 이름 | API/수동 입력 | N/A |
| **원정팀 (Away Team)** | String | 경기장 원정 팀 이름 | API/수동 입력 | N/A |
| **경기 날짜 (Date)** | Date | 경기 일자 | API/수동 입력 | N/A |
| **팀 A 승률 (Team A Win %)** | Float | 최근 N경기(예: 10경기) 승리 비율 | API/데이터 분석 | 높음 |
| **팀 B 승률 (Team B Win %)** | Float | 최근 N경기(예: 10경기) 승리 비율 | API/데이터 분석 | 높음 |
| **홈 이점 지표 (Home Field Advantage)** | Float | 홈 경기 시 평균 승률 보정치 | 데이터 분석 | 중간 |
| **상대 전적 (H2H Record)** | Tuple (W-L) | 홈팀 vs 원정팀의 최근 전적 (승-패) | 데이터 분석 | 높음 |
| **선발 투수 컨디션 (Pitcher Status)** | Float | 선발 투수의 평균 ERA, 투구 수 등 | API/데이터 분석 | 매우 높음 |
| **최근 득점력 (Recent Scoring)** | Float | 최근 5경기 평균 득점 점수 | API/데이터 분석 | 중간 |
| **기상 조건 (Weather)** | String | 비, 맑음, 바람 등 | 외부 API (Weather API) | 낮음 |

### 3. 데이터 확보 및 전처리 전략 (Data Pipeline)

데이터를 안정적으로 확보하고 분석 가능한 형태로 가공하는 과정이 필수적입니다.

1. **데이터 수집 (Collection):**
    * **API 연동:** 전문 스포츠 데이터 API (예: Sportradar, Opta)를 통해 실시간 경기 결과 및 상세 스탯을 수집합니다.
    * **크롤링:** 특정 통계치(예: 특정 팀의 특정 선수 기록)는 정기적인 크롤링을 고려합니다.
2. **데이터 정제 (Cleaning):**
    * **결측치 처리:** 데이터가 누락된 경우, 직전 경기 평균값 또는 중앙값으로 대체합니다.
    * **표준화:** 팀명, 선수명 등 텍스트 데이터의 오타 및 표기법을 통일합니다.
3. **데이터 가공 (Feature Engineering):**
    * **지표 생성:** 단순 통계(승률) 외에 **'최근 5경기 평균 득점 차', '홈 이점 가중치'** 등 예측에 유용한 새로운 지표를 수학적으로 계산하여 추가합니다.

---

### 4. 후속 작업 및 활용 방안 (Prediction Model)

수집된 데이터를 바탕으로 최종적으로 예측 모델을 구동합니다.

1. **모델링:** **로지스틱 회귀(Logistic Regression)** 또는 **XGBoost**와 같은 머신러닝 모델을 사용하여, 위에서 정의된 모든 지표를 독립 변수로, 승패(Binary)를 종속 변수로 설정하여 확률을 산출합니다.
2. **가중치 조정:** 초기 모델링 후, 실제 경기 결과와 모델의 예측 오차를 비교하여 각 지표(예: '선발 투수 컨디션'의 가중치)의 중요도를 지속적으로 조정하는 **반복 학습(Iterative Learning)** 과정이 필수적입니다.
