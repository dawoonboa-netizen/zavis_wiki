## 주식 데이터 API 연결성 및 데이터 파이프라인 요구사항 분석

CEO의 지시에 따라, '주식 트레이더 웹사이트' 구축을 위한 데이터 API 연결 가능성, 필요한 데이터 소스, 그리고 데이터 파이프라인 요구사항을 기술적 타당성을 기반으로 정리합니다.

### 1. 데이터 소스 및 API 타당성 조사

웹사이트의 핵심 기능(실시간 시각화, 백테스팅)을 구현하기 위해 필요한 데이터 유형과 이를 제공할 수 있는 API를 조사했습니다.

| 데이터 유형 | 세부 요구사항 | 추천 데이터 소스 (API 예시) | 기술적 타당성 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **과거 데이터 (Historical Data)** | 일봉, 분봉, 틱 데이터 (최소 5년 이상) | Alpha Vantage, Polygon.io, Yahoo Finance API (스크래핑 대체) | **높음** | 대부분의 금융 데이터 제공업체가 API를 제공하며, 데이터 정합성 확인 필요. |
| **실시간 데이터 (Real-time Data)** | 현재 시세, 실시간 차트 업데이트 | WebSocket 기반 API (예: Polygon.io Streaming, Interactive Brokers API) | **중간** | 실시간 데이터는 지연 시간(Latency)과 API 비용을 고려해야 함. |
| **기초 데이터 (Fundamental Data)** | 재무제표, 기업 뉴스, 산업 데이터 | Financial Modeling Prep (FMP), SEC EDGAR (직접 연동 시) | **중간** | 정형 데이터는 API로 확보 가능하나, 정제 및 연동에 추가 로직 필요. |

**기술적 타당성 요약:**
API 연결 자체는 **매우 타당**합니다. 다만, **실시간 데이터의 안정성**과 **과거 데이터의 정합성** 확보가 성공적인 시스템 구축의 핵심입니다. 무료 API의 경우 데이터 속도(Rate Limit)와 데이터 품질을 신중하게 검토해야 합니다.

### 2. 데이터 파이프라인 요구사항 정의

데이터 수집부터 웹사이트 사용자에게 제공되기까지의 전체 흐름을 위한 파이프라인 요구사항을 정의합니다.

#### A. 데이터 수집 및 통합 단계 (Ingestion)
1. **API 연결 모듈 개발:** 선택된 외부 API(예: Polygon.io)에 대한 인증 및 요청 처리 모듈을 개발합니다.
2. **데이터 정규화:** 서로 다른 API에서 수집된 데이터 포맷(Timestamp, OHLCV 등)을 웹사이트가 요구하는 단일 표준 포맷으로 변환하는 ETL(Extract, Transform, Load) 로직을 구축합니다.
3. **실시간 스트리밍 구축:** 실시간 데이터(WebSocket)를 안정적으로 수신하고, 이를 웹사이트의 프론트엔드에 지연 없이 전달하기 위한 메시지 브로커(예: Kafka, Redis Pub/Sub) 구성을 고려합니다.

#### B. 데이터 저장 및 관리 단계 (Storage)
1. **시계열 데이터베이스 (Time-Series DB):** 과거의 시계열 데이터를 효율적으로 저장하고 빠른 범위 검색을 지원하기 위해 **InfluxDB** 또는 **PostgreSQL (TimescaleDB 확장)** 사용을 권장합니다.
2. **메타데이터 저장소:** 사용자 설정된 전략(백테스팅 결과), API 키, 사용자 설정 등의 메타데이터는 **PostgreSQL**에 저장하여 관계형으로 관리합니다.

#### C. 데이터 제공 및 서비스 단계 (Delivery)
1. **백엔드 API 구축:** 저장된 데이터를 요청에 따라 필터링하고 집계하여 프론트엔드에 제공하는 RESTful API를 구축합니다.
2. **캐싱 전략:** 자주 요청되는 과거 데이터(예: 일간/주간 데이터)는 **Redis**를 사용하여 캐싱함으로써 데이터베이스 부하를 줄이고 응답 속도를 극대화합니다.

### 3. 최종 데이터 파이프라인 요약 (실행 단계)

$$\text{외부 API} \xrightarrow{\text{Ingestion/Transform}} \text{Redis (Cache)} \xrightarrow{\text{Storage}} \text{TimescaleDB/PostgreSQL} \xrightarrow{\text{API}} \text{Web Frontend}$$

**실행 우선순위:**
1. **데이터 소스 최종 선정:** 초기 MVP를 위해 가장 안정적이고 비용 효율적인 API 1~2개를 선정합니다.
2. **백테스팅 데이터 확보:** 웹사이트의 핵심인 백테스팅을 위해 최소 5년 치의 고품질 과거 데이터를 우선적으로 확보하는 데 리소스를 집중합니다.
3. **실시간 테스트:** 실시간 데이터 스트리밍 테스트를 별도로 진행하여 지연 시간을 측정하고 파이프라인의 병목 지점을 확인합니다.
