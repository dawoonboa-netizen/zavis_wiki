## 시장 트렌드, 주요 지표 및 투자 전략 수립을 위한 핵심 데이터 소스 및 분석 프레임워크 조사 요약

본 보고서는 '주식 트레이더' 구축을 위한 기초 데이터 확보 및 분석 프레임워크를 정의하는 것을 목표로 합니다. 성공적인 투자 전략은 신뢰할 수 있는 데이터 소스, 핵심 지표, 그리고 체계적인 분석 방법론의 결합에서 비롯됩니다.

### 1. 핵심 데이터 소스 (Data Sources)

투자 전략 수립에 필요한 데이터는 크게 거시 경제 데이터, 시장 데이터, 개별 종목 데이터 세 가지로 분류됩니다.

| 데이터 유형 | 주요 데이터 소스 (Source) | 활용 목적 | 신뢰도 고려 사항 |
| :--- | :--- | :--- | :--- |
| **거시 경제 지표** | 각국 중앙은행 데이터 (Fed, ECB, 한국은행), OECD 통계, IMF 데이터 | 금리, 인플레이션, GDP 성장률 등 시장 전반의 방향성 파악 | 공식 발표 자료의 시차 및 해석 차이 주의 |
| **시장 및 자산 데이터** | Bloomberg Terminal, Refinitiv Eikon, Yahoo Finance API, 각 거래소 데이터 | 실시간 시세, 거래량, 시장 심리 지표 파악 | 데이터 피드 품질 및 실시간성 확인 필수 |
| **개별 종목 데이터** | 증권사 HTS/MTS 데이터, 금융 데이터 제공 플랫폼 (예: FnGuide, Alpha Vantage) | 재무제표, 밸류에이션 지표, 기업 공시 정보 | 데이터의 정합성 및 최신성 검증 필요 |
| **뉴스 및 정성 데이터** | 주요 경제 뉴스 매체 (FT, WSJ, 블룸버그), 기업 IR 자료, 공시 자료 | 기업의 비즈니스 변화, 경영진의 발언, 산업 트렌드 이해 | 정보의 편향성(Bias)을 인지하고 교차 검증 필요 |

### 2. 핵심 분석 지표 (Key Indicators)

트레이딩 전략을 구체화하기 위해 반드시 모니터링해야 할 핵심 지표는 다음과 같습니다.

#### A. 거시 경제 지표 (Macro Indicators)
1. **금리 지표:** 기준금리, 장단기 금리차 (실질 금리 변화는 자금 흐름에 결정적 영향을 미침)
2. **인플레이션 지표:** 소비자물가지수(CPI), 생산자물가지수(PPI) (실물 경제의 건전성 판단)
3. **성장 지표:** GDP 성장률, 고용 보고서 (경제 사이클 파악)

#### B. 시장 심리 지표 (Market Sentiment Indicators)
1. **VIX 지수 (공포 지수):** 시장의 변동성 및 공포 수준 측정 (높을수록 위험 선호 심리)
2. **주가 변동성:** 특정 기간(5일, 20일)의 평균 변동폭 분석 (단기 시장의 과열/침체 판단)
3. **거래량:** 가격 움직임의 신뢰도 확인 (거래량이 동반되지 않은 가격 변동은 신중해야 함)

#### C. 개별 종목 지표 (Fundamental & Technical Indicators)
1. **재무 건전성:** 매출액, 영업이익, 순이익, 부채비율 (기업의 내재 가치 판단)
2. **밸류에이션:** P/E Ratio (주가수익비율), P/B Ratio (주가순자산비율), PSR (주가매출비율) (상대적 가치 평가)
3. **기술적 분석 (TA):** 이동평균선(MA), RSI (상대강도지수), MACD (이동평균 수렴/확산 지표) (매수/매도 시점 포착)

### 3. 분석 프레임워크 (Analysis Frameworks)

데이터를 바탕으로 실제 투자 결정을 내리기 위한 논리적 구조를 확립해야 합니다.

#### A. Top-Down 분석 (거시 $\rightarrow$ 미시)
*   **단계 1: 거시 환경 분석 (Macro View):** 금리, 인플레이션, 정책 방향을 분석하여 시장의 전반적인 리스크와 기회를 식별합니다.
*   **단계 2: 산업 분석 (Sector View):** 거시 환경 변화에 따라 영향을 받는 산업 섹터를 분석하고, 해당 섹터의 성장 동력과 경쟁 구도를 파악합니다.
*   **단계 3: 개별 종목 분석 (Micro View):** 선택된 섹터 내에서 재무, 기술적 지표를 분석하여 개별 종목의 내재 가치와 매수/매도 시점을 결정합니다.

#### B. Bottom-Up 분석 (미시 $\rightarrow$ 거시)
*   **단계 1: 종목 선정 (Stock Selection):** 개별 기업의 펀더멘털(재무 상태)과 경쟁 우위를 분석하여 투자할 종목군을 선정합니다.
*   **단계 2: 리스크 측정 (Risk Assessment):** 선정된 종목에 대해 잠재적 리스크(재무 위험, 규제 위험, 시장 위험)를 정량화합니다.
*   **단계 3: 포트폴리오 최적화 (Portfolio Optimization):** 자산 간의 상관관계를 분석하여 분산 투자 효과를 극대화하고, 목표 수익률 대비 리스크 수준을 조정하여 최종 포트폴리오를 구성합니다.

### 4. 결론 및 실행 제언

**핵심 전략:** **'시스템 구축'**에 집중해야 합니다. 감정적 대응을 배제하고, **'Bottom-Up 분석'**을 통해 종목을 선정한 후, **'Top-Down 분석'**으로 시장 환경을 교차 검증하는 순환 구조를 확립해야 합니다.

# 📈 초기 투자 전략 프레임워크 (Initial Investment Strategy Framework)

본 프레임워크는 JAY CORP의 투자 활동을 **수익화 모델 검증 및 위험 관리**라는 비즈니스 목표에 맞추어 시스템화하기 위해 정의됩니다. 모든 결정은 아래의 규칙에 따라 이루어져야 합니다.

## 1. 수익 목표 및 위험 허용 범위 (Profit Targets & Risk Tolerance)

투자 활동의 목표와 감수할 수 있는 손실의 한계를 명확히 설정합니다.

| 지표 | 설정 값 (초기 가설) | 근거 및 설명 |
| :--- | :--- | :--- |
| **목표 수익률 (Target Return)** | 연평균 15% (시장 대비 초과 수익 목표) | 장기적인 자본 성장을 목표로 하되, 시장 변동성을 고려하여 공격적이지 않은 목표 설정. |

| **기대 수익 대비 위험 (Risk/Reward Ratio)** | 최소 1:2 이상 | 진입 시 예상 수익이 최대 손실 위험보다 최소 2배 이상 크다는 것을 보장해야 한다. |

## 2. 투자 스타일 정의 (Investment Style Definition)

선택된 분석 방법론과 시간 프레임을 정의합니다.

*   **분석 스타일:** **기술적 분석 (Technical Analysis) 기반의 기본적 분석 (Fundamental Analysis) 결합**
    *   **기술적 분석:** 시장의 모멘텀, 지지/저항 레벨, 거래량(Volume)을 사용하여 진입 및 청산 시점을 결정한다. (실행 중심)
    *   **기본적 분석:** 기업의 재무 건전성(매출, 이익 성장률) 및 산업 트렌드(미래 성장 동력)를 분석하여 투자 대상의 내재적 가치를 확인한다. (선별 중심)
*   **시간 프레임:** **스윙 트레이딩 (Swing Trading)**
    *   **목표:** 며칠에서 몇 주 동안의 가격 변동을 이용하여 수익을 실현한다. (단기 매매의 변동성 회피)

## 3. 리스크 관리 규칙 (Risk Management Rules)

모든 거래에 적용되는 비상 정지(Kill Switch) 규칙입니다.

1.  **포지션 사이징 (Position Sizing):**
    *   **규칙:** 단일 종목에 투자하는 최대 금액은 전체 투자 자산의 **5%**를 초과해서는 안 된다.
    *   **근거:** 최대 손실 허용 범위(-20%)를 고려하여, 한 번의 실패가 전체 자본에 미치는 영향을 통제한다.
2.  **손절매 (Stop-Loss):**
    *   **규칙:** 모든 진입 시점에 **명확한 손절매(Stop-Loss) 가격**을 설정하고, 이는 **기술적 지지선(Support Level)** 또는 **기본적 가치 훼손 시점**에 설정한다.
    *   **실행:** 손절매는 감정 없이 기계적으로 실행한다.
3.  **포트폴리오 분산 (Diversification):**
    *   **규칙:** 투자 자산을 최소 **3개 이상의 상이한 산업 섹터**에 분산 투자하여 단일 산업 리스크를 회피한다.
4.  **비상 정지 (Kill Switch):**
    *   **규칙:** 포트폴리오 전체 손실이 **-20%**에 도달하거나, 설정된 최대 손실 허용 범위를 초과할 경우, **모든 신규 거래를 즉시 중단**하고 현금 비중을 70% 이상으로 전환한다.

## 📊 초기 트레이딩 시스템 아키텍처 설계 (Data Pipeline & Backtesting Framework)

수립된 투자 전략(스윙 트레이딩, 기술적/기본적 분석 결합, 1:2 리스크/보상)을 구현하기 위한 데이터 수집, 처리, 백테스팅 환경의 초기 아키텍처를 설계합니다. 이 설계는 실행 가능한 Python 기반의 모듈 구조를 목표로 합니다.

### 1. 시스템 아키텍처 개요 (High-Level Architecture)

시스템은 **데이터 수집 계층(Data Ingestion)**, **데이터 처리 계층(Data Processing)**, **전략 실행 계층(Strategy Execution)**, **결과 보고 계층(Reporting)**의 4단계로 구성됩니다.

```mermaid
graph TD
    A[외부 데이터 소스: API/DB] --> B{Data Ingestion Layer};
    B --> C[Data Storage: Time-Series DB (e.g., PostgreSQL/Parquet)];
    C --> D{Data Processing Layer: Indicator Calculation};
    D --> E[Strategy Layer: Signal Generation];
    E --> F[Backtesting Engine];
    F --> G[Reporting & Visualization];
```

### 2. 데이터 수집 및 연동 구조 (Data Ingestion & API Linkage)

**목표:** 신뢰성 있고 안정적인 시계열 데이터(가격, 거래량)를 확보합니다.

#### 2.1. API 연동 모듈 정의
외부 데이터 소스(예: Yahoo Finance API, 증권사 API)에 대한 접근을 관리하는 인터페이스를 정의합니다.

*   **모듈:** `data_fetcher.py`
*   **기능:** 특정 종목 코드와 기간을 입력받아 원시 가격 데이터(OHLCV)를 요청하고 JSON/CSV 형태로 반환합니다.
*   **보안:** 모든 API 키 및 인증 정보는 환경 변수(`os.environ`)를 통해 주입받으며, 코드 내에 하드코딩하지 않습니다.

#### 2.2. 데이터 저장 구조 (Data Storage)
과거 데이터의 효율적인 조회를 위해 시계열 데이터베이스 또는 Parquet 파일을 사용합니다.

*   **저장 방식:** 시계열 데이터는 `Timestamp`, `Open`, `High`, `Low`, `Close`, `Volume` 필드를 포함하는 **Parquet 파일** 형태로 저장합니다.
*   **장점:** 대용량 데이터 처리에 효율적이며, 백테스팅 시 데이터 로딩 속도를 최적화합니다.

### 3. 데이터 처리 및 전략 로직 (Data Processing & Strategy Logic)

**목표:** 원시 데이터를 분석 지표로 변환하고, 정의된 투자 스타일을 적용하여 매매 신호를 생성합니다.

#### 3.1. 지표 계산 모듈 정의
연구에서 정의된 지표(VIX, 이동평균선, 볼린저 밴드 등)를 계산하는 함수를 모듈화합니다.

*   **모듈:** `indicator_calculator.py`
*   **기능:** 저장된 OHLCV 데이터를 입력받아, 기술적 분석에 필요한 모든 지표(예: 5일/20일 이동평균선, RSI, MACD)를 계산하여 데이터셋에 추가합니다.

#### 3.2. 트레이딩 로직 (Signal Generation)
실제 매매 결정을 내리는 핵심 로직을 구현합니다.

*   **모듈:** `strategy_engine.py`
*   **로직:**
    1.  **필터링:** 거시 경제 데이터(선택 사항) 또는 시장 심리 지표를 기반으로 전체 포트폴리오의 위험 수준을 점검합니다.
    2.  **신호 생성:** `indicator_calculator`의 결과를 바탕으로 **진입(Buy/Sell)** 및 **청산(Exit)** 신호를 생성합니다.
    3.  **규칙 적용:** 리스크 관리 규칙(포지션 사이징, 손절매 레벨)을 신호에 적용하여 최종 실행 가능한 거래 명령을 출력합니다.

### 4. 백테스팅 환경 구축 (Backtesting Environment)

**목표:** 실제 시장 데이터가 아닌 과거 데이터를 사용하여 전략의 성과를 검증합니다.

*   **모듈:** `backtester.py`
*   **기능:**
    1.  **데이터 로드:** 저장소에서 필요한 과거 데이터를 로드합니다.
    2.  **시뮬레이션:** `strategy_engine`에서 생성된 신호를 시간 순서대로 따라가며 가상의 거래를 실행합니다.
    3.  **성과 측정:** 거래 시점별 수익률, 최대 손실 폭(Max Drawdown), 연평균 수익률(CAGR), 승률(Win Rate) 등을 계산하여 결과를 산출합니다.
    4.  **위험 검증:** 백테스팅 결과가 설정된 최대 허용 손실(-20%) 규칙을 위반했는지 검증합니다.

### 2. 데이터 파이프라인 요구사항 정의

데이터 수집부터 웹사이트 사용자에게 제공되기까지의 전체 흐름을 위한 파이프라인 요구사항을 정의합니다.

#### A. 데이터 수집 및 통합 단계 (Ingestion)
1. **API 연결 모듈 개발:** 선택된 외부 API(예: Polygon.io)에 대한 인증 및 요청 처리 모듈을 개발합니다.
2. **데이터 정규화:** 서로 다른 API에서 수집된 데이터 포맷(Timestamp, OHLCV 등)을 웹사이트가 요구하는 단일 표준 포맷으로 변환하는 ETL(Extract, Transform, Load) 로직을 구축합니다.
3. **실시간 스트리밍 구축:** 실시간 데이터(WebSocket)를 안정적으로 수신하고, 이를 웹사이트의 프론트엔드에 지연 없이 전달하기 위한 메시지 브로커(예: Kafka, Redis Pub/Sub) 구성을 고려합니다.

#### B. 데이터 저장 및 관리 단계 (Storage)
1. **시계열 데이터베이스 (Time-Series DB):** 과거의 시계열 데이터를 효율적으로 저장하고 빠른 범위 검색을 지원하기 위해 **InfluxDB** 또는 **PostgreSQL (TimescaleDB 확장)** 사용을 권장합니다.
2. **메타데이터 저장소:** 사용자 설정된 전략(백테스팅 결과), API 키, 사용자 설정 등의 메타데이터는 **PostgreSQL**에 저장하여 관계형으로 관리합니다.

#### C. 데이터 제공 및 서비스 단계 (Delivery)
1. **백엔드 API 구축:** 저장된 데이터를 요청에 따라 필터링하고 집계하여 프론트엔드에 제공하는 RESTful API를 구축합니다.
2. **캐싱 전략:** 자주 요청되는 과거 데이터(예: 일간/주간 데이터)는 **Redis**를 사용하여 캐싱함으로써 데이터베이스 부하를 줄이고 응답 속도를 극대화합니다.

### 1.2. 백엔드 (Backend: 데이터 처리 및 API 제공)
*   **기술:** **Node.js (Express/NestJS)** 또는 **Python (FastAPI)**
    *   **선택 이유:** 금융 데이터 파이프라인의 복잡성을 관리하고, 특히 **WebSocket**을 통한 실시간 스트리밍 통신을 효율적으로 처리하기에 적합합니다. (Python은 데이터 분석 라이브러리 연동에 강점, Node.js는 실시간 I/O 처리에 강점)
*   **실시간 통신:** **WebSocket** (실시간 시세 스트리밍)
*   **비동기 작업 처리:** **Redis** (캐싱 및 세션 관리), **BullMQ/Celery** (백테스팅 및 대규모 데이터 처리 작업 큐)

| **사용자 데이터** | 보안 및 무결성 | 사용자 설정, 거래 기록 | 관계형 DB (PostgreSQL) |

#### 3.2. 기술 아키텍처 제안 (Proposed Architecture)

| 계층 (Layer) | 주요 기술 스택 | 역할 및 데이터 흐름 |
| :--- | :--- | :--- |
| **프론트엔드 (Client)** | React.js / TypeScript | 사용자 인터페이스, 차트 시각화, 사용자 입력 처리. WebSocket을 통한 실시간 데이터 수신. |
| **백엔드 (Server)** | Node.js (Express/NestJS) | API 엔드포인트 제공, 사용자 인증 및 권한 관리, WebSocket 서버 운영. |
| **데이터베이스 (DB)** | PostgreSQL (또는 TimescaleDB) | 사용자 정보, 거래 기록 등 관계형 데이터의 안정적 저장. |
| **데이터 파이프라인** | Kafka / Redis | 실시간 데이터 스트리밍 및 캐싱, 대용량 시계열 데이터의 효율적 관리. |
